# MCP capabilities

- **Prod source lives under `src/`** (ADR 0008): `src/app` (routes + `src/proxy.ts` middleware), `src/components`, `src/lib`. The `@/*` alias maps to `./src/*` (tsconfig + vitest), so imports stay `@/lib/‚Ä¶`. Config, `public/`, `supabase/`, `scripts/`, `test/`, and `legal/` stay at the `code/` root.
- **Every MCP tool registers through `registerGatedTool`** (`src/lib/mcp/register-gated-tool.ts`) so it's metered by the billing gate (ADR 0002) ‚Äî never call `server.tool` directly for a capability. `tools.ts` is a thin composer; each capability lives in `src/lib/mcp/skills/` (procedure codificate) or `src/lib/mcp/loops/` (area 09). Prompts (method/template, ungated) go in `src/lib/mcp/prompts.ts`.
- **Telemetry is free for gated tools** (ADR 0006): `registerGatedTool` records one `tool_events` row per call (tool, outcome, latency, argument NAMES ‚Äî never values) via `src/lib/mcp/telemetry.ts`, fail-open. The single ungated exception is `invia_feedback` (`src/lib/mcp/feedback.ts`), which must keep working past the paywall; don't add others.
- **Pure fiscal logic lives in `src/lib/fiscal/`**, not in the tool handlers ‚Äî funzioni pure e testabili. Constants with legal weight live in the **registry** (`src/lib/fiscal/constants.ts` via `registry.ts`, ADR 0011): every constant is `{ valore, fonte, verificatoIl }`; time-varying values are vigenza-indexed tables read via `lookupVigente` by date ‚Äî never add a bare `export const` with legal weight, and never overwrite historical vigenze (append). Out-of-range lookups return `verificato: false` with a `nota` the tool MUST surface in its output. Re-verification: `/verifica-costanti` skill; every output is a *bozza* (human-in-the-loop). Italian format parsers are in `src/lib/parse/`.
- **No server-side domain state** (ADR 0003): the MCP server never persists studio/client fiscal data ‚Äî that stays in the studio's files, managed by the client (Claude Code) under the `studio/` markdown convention (ADR 0010; source of truth: the `convenzione_studio_db` prompt in `src/lib/mcp/prompts.ts`). Tools with state must say in their output which `studio/` file to update. Supabase holds billing + product signal only (`tool_events`, `feedback` ‚Äî ADR 0006), never fiscal data.
- **DB schema changes go through the declarative schema** (ADR 0007): edit `supabase/schemas/*.sql` (the source of truth), then `npm run db:diff -- -f <name>` to generate the migration and `npm run db:push` to apply ‚Äî never hand-write migrations or run SQL in the Supabase editor. Regenerate `src/lib/billing/database.types.ts` with `npm run db:types`. Runbook: `content/knowledge/db-schema-migrations.md`.
- **Tests:** `npm run test` (vitest). Cover the arithmetic in `src/lib/fiscal/` and `src/lib/parse/` when you touch it.
