# Dott. Comm. knowledge

Dott. Comm.'s settled, current truth — its own build and comms reference. Each doc
uses frontmatter (`title`, `status`, `owner`, `updated`, `tags`); start from
[`templates/knowledge-doc.md`](../../../../templates/knowledge-doc.md).

Organize into subfolders as content grows (e.g. `engineering/`, `comms/`,
`clients/`, `product/`). Keep it current; group-wide knowledge lives in
`group/content/knowledge/` instead.
